88  // GEOMETRY
89  //
90  // NOTE: Vertex and Triangle are not object classes because we want to be able to
91  // download them into the GPU without modification. This is a performance
92  // consideration.
93  //
94  // Geom is not an object class for the same reason.
95  // However, Geom is hidden (known only between World.cpp and Sys.h).
96  // Instead, users of this module get back an opaque geometry handle (an int).
97  //
98  // geom_add() returns an opaque handle for referring to the added geometry in subsequent calls.
99  // The vertex and triangle arrays must continue to exist after this call.
100  // They will get copied later, possibly multiple times if changes occur.
101  // For performance reasons, it's important to supply a fairly accurate hint for "changes".
102  //
103  // geom_changed() is called to note that the vertex data has changed, but the number and order
104  // of the vertices and triangles hasn't.
105  //
106  // geom_remove() is called to remove the geometry. Currently, if you want to change the number
107  // or order of vertexes or triangles, you must first remove the old geometry using this function.
108  // This is also called when an Entity is deleted.
109  //
110  int geom_add( Vertex * vertex, int vertex_cnt, Triangle * triangle, int triangle_cnt, int changes = GEOM_CHANGES_RARELY );
